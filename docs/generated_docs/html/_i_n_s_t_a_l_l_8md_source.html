27 <a name="unix" />
28 ###Most Unix like OSes:
29 
30 #### Quick install:
31 
32 On Gentoo:
33 ```
34 # layman -f && layman -a tox-overlay && emerge net-libs/tox
35 ```
36 
37 And you're done `:)`</br>
38 If you happen to run some other distro which isn't made for compiling, there are steps below:
39 
40 #### Build manually
41 
42 Build dependencies:
43 
44 Note: package fetching commands may vary by OS.
45 
46 On Ubuntu `< 15.04` / Debian `< 8`:
47 
48 ```bash
49 sudo apt-get install build-essential libtool autotools-dev automake checkinstall check git yasm
50 ```
51 
52 On Ubuntu `>= 15.04` / Debian `>= 8`:
53 ```bash
54 sudo apt-get install build-essential libtool autotools-dev automake checkinstall check git yasm libsodium13 libsodium-dev
55 ```
56 
57 On Fedora:
58 
59 ```bash
60 yum groupinstall "Development Tools"
61 yum install libtool autoconf automake check check-devel
62 ```
63 
64 On SunOS:
65 
66 ```pfexcec
67 pkg install autoconf automake gcc-47
68 ```
69 On FreeBSD 10+:
70 
71 ```tcsh
72 pkg install net-im/tox
73 ```
74 Note, if you install from ports select NaCl for performance, and sodium if you want it to be portable.
75 
76 **For A/V support, also install the dependences listed in the [libtoxav] (#libtoxav) section.**
77 
78 You should get and install [libsodium](https://github.com/jedisct1/libsodium). If you have installed `libsodium` from repo, ommit this step, and jump directly to [compiling toxcore](#compile-toxcore):
79 ```bash
80 git clone git://github.com/jedisct1/libsodium.git
81 cd libsodium
82 git checkout tags/1.0.3
83 ./autogen.sh
84 ./configure && make check
85 sudo checkinstall --install --pkgname libsodium --pkgversion 1.0.0 --nodoc
86 sudo ldconfig
87 cd ..
88 ```
89 
90 
91 Or if checkinstall is not easily available for your distribution (e.g., Fedora),
92 this will install the libs to /usr/local/lib and the headers to /usr/local/include:
93 
94 ```bash
95 git clone git://github.com/jedisct1/libsodium.git
96 cd libsodium
97 git checkout tags/1.0.3
98 ./autogen.sh
99 ./configure
100 make check
101 sudo make install
102 cd ..
103 ```
104 
105 If your default prefix is /usr/local and you happen to get an error that says "error while loading shared libraries: libtoxcore.so.0: cannot open shared object file: No such file or directory", then you can try running ```sudo ldconfig```. If that doesn't fix it, run:
106 ```
107 echo '/usr/local/lib/' | sudo tee -a /etc/ld.so.conf.d/locallib.conf
108 sudo ldconfig
109 ```

118 ./configure
119 make
120 sudo make install
121 ```

126 
127 You need the latest XCode with the Developer Tools (Preferences -> Downloads -> Command Line Tools).
128 The following libraries are required along with libsodium and cmake for Mountain Lion and XCode 4.6.3 install libtool, automake and autoconf. You can download them with Homebrew, or install them manually.

184 Macports: (https://www.macports.org/)
185 All toxcore dependencies can be installed from MacPorts. This is often easier on PowerPC Macs,
186 and any version of OS X prior to 10.6, since Homebrew is supported on 10.6 and up, but not much
187 (or at all) on older systems. A few packages have slightly different names from the corresponding
188 package in Debian.
189 
190 Same: libtool autoconf automake libsodium check yasm
191 Different: libvpx (webm) libopus pkgconfig gettext
192 
193 (the libintl, from gettext, built into OS X 10.5 is missing libintl_setlocale, but the Macports build has it)
194 
195 Verify where libintl is on your system: (MacPorts puts it in /opt/local)
196 ```
197 for d in /usr/local/lib /opt/local/lib /usr/lib /lib; do ls -l $d/libintl.*; done
198 ```
199 
200 Check if that copy has libintl_setlocale:
201 ```
202 nm /opt/local/lib/libintl.8.dylib | grep _libintl_setlocale
203 ```
204 
205 Certain other tools may not be installed, or outdated, and should also be installed from MacPorts for simplicity: git cmake
206 
207 If libsodium was installed with MacPorts, you may want to symlink the copy in /opt/local/lib to /usr/local/lib. That way you don't need special configure switches for toxcore to find libsodium, and every time MacPorts updates libsodium, the new version will be linked to toxcore every time you build:
208 ```
209 ln -s /opt/local/lib/libsodium.dylib /usr/local/lib/libsodium.dylib
210 ```
211 
212 Much of the build can then be done as for other platforms: git clone, and so on. Differences will be noted with (OS X 10.5 specific)
213 
214 pkg-config is important for enabling a/v support in tox core, failure to install pkg-config will prevent tox core form finding the required libopus/libvpx libraries. (pkg-config may not configure properly, if you get an error about GLIB, run configure with the following parameter, --with-internal-glib).
215 
216 Uncompress and install them all. Make sure to follow the README as the instructions change, but they all follow the same pattern below:
217 
218 ```bash
219 ./configure
220 make
221 sudo make install
222 ```

227 cd toxcore

229 ./configure (OS X 10.5 specific)
230 ./configure CC="gcc -arch ppc -arch i386" CXX="g++ -arch ppc -arch i386" CPP="gcc -E" CXXCPP="g++ -E"
231 make
232 make install (OS X 10.5 specific)
233 should be: sudo make install
234 If it worked, you should have all the toxcore dylibs in /usr/local/lib: (besides the four below, the rest are symlinks to these)
235 $ ls -la /usr/local/lib/libtox*.dylib

242 You should now be able to move on to compiling Toxic/Venom or some other client application
243 There is also a shell script called "osx_build_script_toxcore.txt" which automates everything from "git pull" to "sudo make install", once the dependencies are already taken care of by MacPorts.
244 ```

248 ```
249 ./configure --with-libsodium-headers=/usr/local/include/ --with-libsodium-libs=/usr/local/lib
250 ```

265 ####Cross-compile
266 
267 It's a bit challenging to build Tox and all of its dependencies nativly on Windows, so we will show an easier, less error and headache prone method of building it -- cross-compiling.
268 
269 <a name="windows-cross-compile-vm" />
270 #####Setting up a VM
271 
272 We will assume that you don't have any VM running Linux around and will guide you from the ground up.
273 
274 First, you would need to get a virtual machine and a Linux distribution image file.
275 
276 For a virtual machine we will use VirtualBox. You can get it [here](https://www.virtualbox.org/wiki/Downloads).
277 
278 For a Linux distribution we will use Lubuntu 14.04 32-bit, which you can get [here](https://help.ubuntu.com/community/Lubuntu/GetLubuntu).
279 
280 After you have those downloaded, install the VirtualBox and create a VM in it. The default of 512mb of RAM and 8gb of dynamically-allocated virtual hard drive would be enough.
281 
282 When you have created the VM, go into its **Settings** -> **System** -> **Processor** and add some cores, if you have any additional available, for faster builds.
283 
284 Then, go to **Settings** -> **Storage**, click on **Empty** under **Controller: IDE**, click on the little disc icon on the right side of the window, click on **Choose a virtual CD/DVD disk file** and select the downloaded Lubuntu image file.
285 
286 Start the VM and follow the installation instructions.
287 
288 After Lubuntu is installed and you have booted into it, in VirtualBox menu on top of the window select **Devices** -> **Insert Guest Additions CD image...**.
289 
290 Open terminal from **Lubuntu's menu** -> **Accessories**.
291 
292 Execute:
293 ```bash
294 sudo apt-get update
295 sudo apt-get install build-essential -y
296 cd /media/*/*/
297 sudo ./VBoxLinuxAdditions.run
298 ```
299 
300 After that, create a folder called `toxbuild` somewhere on your Windows system. The go to **Devices** -> **Shared Folders Settings...** in the VirtualBox menu, add the `toxbuild` folder there and set **Auto-mount** and **Make Permanent** options.
301 
302 Execute:
303 ```bash
304 sudo adduser `whoami` vboxsf
305 ```
306 Note the use of a [grave accent](http://en.wikipedia.org/wiki/Grave_accent) instead of an apostrophe.
307 
308 Then just reboot the system with:
309 ```bash
310 sudo reboot
311 ```
312 
313 After the system is booted, go to **Devices** -> **Shared Clipboard** and select **Bidirectional**. Now you will be able to copy-paste text between the host and the guest systems.
314 
315 Now that the virtual machine is all set up, let's move to getting build dependencies and setting up environment variables.
316 
317 <a name="windows-cross-compile-environment" />
318 #####Setting up the environment
319 
320 First we will install all tools that we would need for building:
321 ```bash
322 sudo apt-get install build-essential libtool autotools-dev automake checkinstall check git yasm pkg-config mingw-w64 -y
323 ```
324 
325 Then we will define a few variables, **depending on which you will build either 32-bit or 64-bit Tox**.
326 
327 For 32-bit Tox build, do:
328 ```bash
329 WINDOWS_TOOLCHAIN=i686-w64-mingw32
330 LIB_VPX_TARGET=x86-win32-gcc
331 ```
332 
333 For 64-bit Tox build, do:
334 ```bash
335 WINDOWS_TOOLCHAIN=x86_64-w64-mingw32
336 LIB_VPX_TARGET=x86_64-win64-gcc
337 ```
338 
339 This is the only difference between 32-bit and 64-bit build procedures.
340 
341 For speeding up the build process do:
342 ```
343 MAKEFLAGS=j$(nproc)
344 export MAKEFLAGS
345 ```
346 
347 And let's make a folder where we will be building everything at
348 ```bash
349 cd ~
350 mkdir prefix
351 cd prefix
352 PREFIX_DIR=$(pwd)
353 cd ..
354 mkdir build
355 cd build
356 ```
357 
358 <a name="windows-cross-compile-compiling" />
359 #####Compiling
360 
361 Now we will build libraries needed for audio/video: VPX and Opus.
362 
363 VPX:
364 ```bash
365 git clone http://git.chromium.org/webm/libvpx.git
366 cd libvpx
367 git checkout tags/v1.3.0
368 CROSS="$WINDOWS_TOOLCHAIN"- ./configure --target="$LIB_VPX_TARGET" --prefix="$PREFIX_DIR" --disable-examples --disable-unit-tests --disable-shared --enable-static
369 make
370 make install
371 cd ..
372 ```
373 
374 Opus:
375 ```bash
376 wget http://downloads.xiph.org/releases/opus/opus-1.1.tar.gz
377 tar -xf opus-1.1.tar.gz
378 cd opus-1.1
379 ./configure --host="$WINDOWS_TOOLCHAIN" --prefix="$PREFIX_DIR" --disable-extra-programs --disable-doc --disable-shared --enable-static
380 make
381 make install
382 cd ..
383 ```
384 
385 Now we will build sodium crypto library:
386 ```bash
387 git clone https://github.com/jedisct1/libsodium/
388 cd libsodium
389 git checkout tags/1.0.0
390 ./autogen.sh
391 ./configure --host="$WINDOWS_TOOLCHAIN" --prefix="$PREFIX_DIR" --disable-shared --enable-static
392 make
393 make install
394 cd ..
395 ```
396 
397 And finally we will build Tox:
398 ```bash
399 git clone https://github.com/irungentoo/toxcore
400 cd toxcore
401 ./autogen.sh
402 ./configure --host="$WINDOWS_TOOLCHAIN" --prefix="$PREFIX_DIR" --disable-ntox --disable-tests --disable-testing --with-dependency-search="$PREFIX_DIR" --disable-shared --enable-static
403 make
404 make install
405 cd ..
406 ```
407 
408 Then we make Tox shared library:
409 ```bash
410 cd "$PREFIX_DIR"
411 mkdir tmp
412 cd tmp
413 $WINDOWS_TOOLCHAIN-ar x ../lib/libtoxcore.a
414 $WINDOWS_TOOLCHAIN-ar x ../lib/libtoxav.a
415 $WINDOWS_TOOLCHAIN-ar x ../lib/libtoxdns.a
416 $WINDOWS_TOOLCHAIN-ar x ../lib/libtoxencryptsave.a
417 $WINDOWS_TOOLCHAIN-gcc -Wl,--export-all-symbols -Wl,--out-implib=libtox.dll.a -shared -o libtox.dll *.o ../lib/*.a /usr/$WINDOWS_TOOLCHAIN/lib/libwinpthread.a -liphlpapi -lws2_32 -static-libgcc
418 ```
419 
420 And we will copy it over to the `toxbuild` directory:
421 ```bash
422 mkdir -p /media/sf_toxbuild/release/lib
423 cp libtox.dll.a /media/sf_toxbuild/release/lib
424 mkdir -p /media/sf_toxbuild/release/bin
425 cp libtox.dll /media/sf_toxbuild/release/bin
426 mkdir -p /media/sf_toxbuild/release/include
427 cp -r ../include/tox /media/sf_toxbuild/release/include
428 ```
429 
430 That's it. Now you should have `release/bin/libtox.dll`, `release/bin/libtox.dll.a` and `release/include/tox/<headers>` in your `toxbuild` directory on the Windows system.

482 ###Advanced configure options:
483 
484  - --prefix=/where/to/install
485  - --with-libsodium-headers=/path/to/libsodium/include/
486  - --with-libsodium-libs=/path/to/sodiumtest/lib/
487  - --enable-silent-rules less verbose build output (undo: "make V=1")
488  - --disable-silent-rules verbose build output (undo: "make V=0")
489  - --disable-tests build unit tests (default: auto)
490  - --disable-av disable A/V support (default: auto) see: [libtoxav](#libtoxav)
491  - --enable-ntox build nTox client (default: no) see: [nTox](#ntox)
492  - --enable-daemon build DHT bootstrap daemon (default=no) see: [Bootstrap daemon](#bootstrapd)
493  - --enable-shared[=PKGS] build shared libraries [default=yes]
494  - --enable-static[=PKGS] build static libraries [default=yes]

498 ###A/V support:
499 
500 <a name="libtoxav" />
501 ####libtoxav:
502 
503 'libtoxav' is needed for A/V support and it's enabled by default. You can disable it by adding --disable-av argument to ./configure script like so:
504 ```bash
505 ./configure --disable-av
506 ```
507 
508 There are 2 dependencies required for libtoxav: libopus and libvpx. If they are not installed A/V support is dropped.
509 
510 Install on fedora:
511 ```bash
512 yum install opus-devel libvpx-devel
513 ```
514 
515 Install on ubuntu:
516 ```bash
517 sudo apt-get install libopus-dev libvpx-dev pkg-config
518 ```
519 If you get the "Unable to locate package libopus-dev" message, add the following ppa and try again:
520 ```bash
521 sudo add-apt-repository ppa:ubuntu-sdk-team/ppa && sudo apt-get update && sudo apt-get dist-upgrade
522 ```
523 
524 Install from source (example for most unix-like OS's):
525 
526 libvpx:
527 ```bash
528 git clone http://git.chromium.org/webm/libvpx.git
529 cd libvpx
530 ./configure
531 make -j3
532 sudo make install
533 cd ..
534 ```
535 
536 libopus:
537 ```bash
538 wget http://downloads.xiph.org/releases/opus/opus-1.0.3.tar.gz
539 tar xvzf opus-1.0.3.tar.gz
540 cd opus-1.0.3
541 ./configure
542 make -j3
543 sudo make install
544 cd ..
